# 戦闘仕様 v1(ターン制・決定論)

2026-07-17の2本のプレイログで確定した方針:**戦闘の進行・結果はすべてシステム(JS)が確定し、LLMは確定済みの結果を描写するだけ**。LLMが応答不能でも戦闘は定型文で完走する。

背景(2026-07-17クロニクルで観測した欠陥):
- 攻撃成功時のダメージをLLMの`enemy_hp_delta`提案に委ねていたため、クリティカルが無効化される
- 判定前後の2回の応答が両方`hp_delta`を出して二重ダメージ
- 小型モデルが同一の語り+同一の台詞を8ターン繰り返し、プレイヤーはHP10→0で死亡
- 「ランタンで照らす」の効果がLLMの気分次第(前回は離脱成功、今回は無反応)

## 今回の範囲(v1)

### データ(スキーマ追加)

campaign.json:
- `player.agility` — プレイヤーの機敏さ(既定 6)
- `companions[].agility` — 同行者の機敏さ

chapter_XX.json の `scenes[].enemy`:
- `agility` — 敵の機敏さ(既定 5)
- `atk` — 命中時のダメージ(既定 1)
- `defenseDc` — 攻撃判定のDC(既定 12)
- `fleeDc` — 逃走判定のDC(既定 10)
- `weakness` — `{ triggers: ["ランタン","光",…], effect: "flee"|"stun", text: "発動時の定型文" }`
  弱点行動は判定なしで必ず効く(発見した対抗策への報酬)。`flee`=敵が退く(戦闘終了)、`stun`=敵はこのターン行動不能

### ターンの流れ

戦闘中(`state.enemy`あり)にプレイヤーが**戦闘行動**を宣言した場合のみ、決定論ルートで1ターンを解決する:

1. **行動順**: agility降順(同値はプレイヤー→同行者→敵)。毎ターン同じ順
2. 順番に解決:
   - **プレイヤー**(宣言から分類):
     - 攻撃(「攻撃」「斬る」等) → d20をプレイヤーが振る vs `defenseDc`。成功=1ダメージ、クリティカル=2、ファンブル=敵の追撃1回。攻撃した時点で敵の正体判明(ウィザードリィ式、従来通り)
     - 防御(「防御」「身を守る」等) → このターンのパーティ被ダメージ-1(防御者1人につき-1、最低0)
     - 逃走(「逃げる」等) → d20 vs `fleeDc`。成功=戦闘終了(敵は倒していない)
     - 弱点行動(`weakness.triggers`に一致) → 判定なしで`effect`発動
   - **同行者**(自動行動。乱数: 攻撃60% / 防御20% / 一言20%):
     - 攻撃 → システムが自動でd20(プレイヤーのタップ不要) vs `defenseDc`。成功=1、クリティカル=2
     - 防御 → 被ダメージ-1に参加
     - 一言 → quirksのmutterを短く発する(行動なし)
   - **敵**: d20≥10で命中 → `atk`−(防御人数)のダメージ(最低0)
3. 敵HP0 → 撃破処理(従来の`checkEnemyDown`: 正体判明・`revealOnDefeat`開示)
4. **語り**: 確定した結果一覧をLLMに渡し「3文以内で描写、数値・新事実の追加禁止」。LLMの`state_updates`は完全無視。応答エラー/タイムアウト時は定型文で代替(進行は止まらない)

### 戦闘行動以外の宣言(調べる・話しかける等)= 「工夫」

戦闘中はLLMの自由裁量ルートを開かない(2026-07-17(2)の亡霊語りループ対策)。定型行動に当てはまらない宣言は、プレイヤーの「工夫」として1ターンを消費して解決する:

- d20 vs DC12 の判定(プレイヤーがダイスを振る)
- 成功し、宣言が現在シーンのsecretに一致すれば真相を開示(戦闘中の調査で正体や弱点に気づく道を維持)
- 一致しなければ「試みた/うまくいかなかった」の定型結果のみ
- 同行者・敵はその後、通常通り行動する

## 将来の範囲(v1では実装しない)

- マップ・パーティ/敵の配置・距離(魔法・飛び道具・防御効果の距離依存はここで一括設計)
- パーティメンバーへの行動指示UI(攻撃/防御/逃げる/委任)。v1は全員「委任」相当の乱数。指示UIは乱数の重みを上書きする形で載せる
- メンバー個別HP・戦闘不能(案B)。v1は共有HP(案A)
- 敗北処理の物語化(村に担ぎ込まれて再開等)。v1は従来通りHP0で終了
- バトル専用画面。v1は現行のチャット+ポップアップのまま(モック段階の割り切り)
- 武器・防具によるダメージ/DC修正

## 変更しないもの

- 奇襲(ambush)の仕組み、エンカウント/正体判明ポップアップ、敵スプライト表示
- 非戦闘時の判定・開示・シーン遷移ロジック
