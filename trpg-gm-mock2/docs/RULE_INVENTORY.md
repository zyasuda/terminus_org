**2026-07-21の教訓**: B1〜B5は「段階2(意図分類カスケード)完了後に不要になるはず」という**推測**で「削除可」と記していたが、実際にコード(`sendAction`のLLM呼び出し経路)を1行ずつ辿ったのはB4のみで、B1〜B3は検証されていなかった。結果、実際に死んでいたのはB4(と、それに同居していたB5の片側)だけだった。
**今後の運用**: 「削除可」の判定は、実装を実際に辿って「このブロックを注入するsystemPrompt呼び出しに、対応する状態(state.enemy/pendingFailedCheck/blockedMove等)が立った状態で到達しうるか」を確認してから記すこと。推測で削除可マークを付けない。

## 意図分類表(段階2の中核)

宣言の解決は常にシステム。分類は「①辞書(高速)→②LLM分類器(穴埋め)→③決定論解決」のカスケード。

| intent | 例 | 解決(決定論) | 状態 |
|---|---|---|---|
| investigate | 調べる/よく見る/聞く/嗅ぐ | secret一致→判定→開示。不一致→LLM描写(状態変更なし) | 実装 |
| move | 奥へ進む | completeRequires検証→遷移 or blockedText | 実装 |
| back | 戻る | 定型(今は戻らない) | 実装 |
| talk | 人物への発言・質問・**平叙文の報告** | LLM会話レーン(語りと同行者のみ。状態変更キーは無視) | 実装 |
| talk_gm | GM(ダイス先輩)への質問・要望 | GM人格で応答。ルール変更はmeta_request扱い | 実装 |
| take | 手に取る/拾う | loot正名照合→所持 or 定型で退場 | 実装 |
| other | 上記以外の工夫 | 非戦闘: LLM描写レーン / 戦闘中: 工夫判定 | 実装 |
| attack/defend/flee/weakness | 戦闘中のみ | tryCombatTurn(完全決定論) | 実装済み |
| (将来) shop / rest / magic | 買い物・休息・魔法 | それぞれの決定論解決をエンジンに足す時に分類も追加 | 未定 |

修飾軸(分類ではない): `actor`(player/gareth/lydia)、`instrument`(ランタン等)。
