# TAS改造・統合出力 引き継ぎ資料

更新日：2026-07-23

## 1. この資料の目的

TAS（ストーリー制作ワークスペース）に対して実施した改造内容、mock2への出力契約、回帰テスト、現在の注意点をまとめます。

TASの最終目的は、キャンペーン・チャプター・シーン・キャラクター・モンスター・アイテム等を編集し、mock2が読み込むJSONへ出力することです。

## 2. 出力経路の整理

ゲーム側へ渡す最終出力は、TAS内の統合処理で確定します。

- `mockCampaignPayloadUnified()`：TASの各入力をゲーム側JSONへ正規化する統合処理
- `window.__tasOutputPipelines.unified()`：統合処理の検証入口
- `window.__tasOutputPipelines.active()`：実運用の出力入口
- `exportPayload()`：AI補助・下書き参照・新規キャンペーン入力用の編集スナップショット。ゲーム側の最終出力ではない

旧11重の`mockCampaignPayload`上書きチェーンは段階的に統合・削除しました。現在は、旧`legacy`比較入口も廃止しています。

## 3. 主な実装内容

### キャンペーンとチャプター

- キャンペーンID、チャプターIDを安定IDとして保持
- mock2が読むネスト構造へ統一
- 出力先は次の形式

```text
public/data/campaigns/<campaignId>/campaign.json
public/data/campaigns/<campaignId>/chapter_01.json
```

- `campaigns.json`のカタログを出力・更新
- キャンペーンのオープニングを`chapter.intro`へ変換
- チャプター構成にイントロ、シーン、アウトロ、インターミッションを反映

### キャラクター・同行者

- 同行者IDを`member_1`、`member_2`、`member_3`形式へ統一
- 配置順をIDへ反映
- 性別属性、一人称、呼称・二人称を出力
- ゲーム側の呼称キーは`addressTerm`へ統一
- 旧`addressing`キーは最終JSONから削除
- 会話掛け合いの参照先も`member_N`へ変換
- 同行者画像の参照先を`campaign.companions.member_N.sprite`へ変換
- ガレスとリディアの表示名とIDが入れ替わらないよう回帰テストを追加

### エンティティ台帳・概念・アイテム

- エンティティ台帳をキャンペーン出力へ統合
- 正式名、別名、種別、重要度、表層説明、確定情報、画像、行動候補を出力
- 重要語・物語概念を`campaign.concepts`へ出力
- アイテムを`campaign.items`へ出力
- 初期所持品を`initialInventory`および`initialInventoryIds`へ出力
- アイテム操作属性（調べる、入手、使う、渡す、紛失など）を保持

### シーン

- 背景画像、SKY画像、前景画像を出力
- シーンNPCをゲーム側の`npc`／`npcSprite`形式へ変換
- シーンの敵本体と`encounters[]`を分離して保持
- 条件付き・任意・ランダム遭遇を出力
- シーン要素、発見情報、入手品、出口・分岐を出力
- 出口条件の入力を日本語タイトル選択式へ改善
- `secretsAny`／`secretsAll`などの完了条件を保持

### フラグとゲーム状態

- `campaign.flags`を出力
- `chapter.flagsOut`を出力
- `chapter.flagRules`を出力
- `style.gameOverText`を出力
- 状態変化ルールと会話ルールの`stateUpdates`を出力

### 画像とmock2連携

- TASで選択した画像をmock2の画像参照へ変換
- `assets`情報にファイル名と使用箇所を出力
- mock2側の出力先は`public/data/campaigns/<campaignId>/`
- 出力後、mock2はキャンペーンカタログを再読み込みする

## 4. mock2側での利用方法

mock2は`public/data/campaigns.json`を読み込み、`defaultCampaign`またはURLのキャンペーンIDを使ってデータを選びます。

例：

```text
?campaign=lanternhill&chapter=chapter_01
```

データJSONを変更しただけなら、開発サーバーでは通常ブラウザの再読み込みだけで反映されます。公開用ビルドを使っている場合はmock2側で`npm run build`が必要です。

キャンペーンを切り替えた後に古いプレイ状態が残る場合は、キャンペーン単位のセーブをリセットして新規開始します。

## 7. 現在の注意点

- 直近のmock2出力で`lanternhill`キャンペーンを生成済みです。
- mock2の`campaigns.json`は`defaultCampaign: "lanternhill"`へ変更されています。
- 旧`campaign`フォルダのJSONも作業ツリー上で変更されています。不要なら、ゲーム側で確認後に整理してください。
- `assets.json`には既存資産の古い`usedBy`情報による警告が残っていますが、アセット検査自体は合格しています。
- 画像を変更した場合は、TASから再度「mock側へ出力」を押し、mock2を再読み込みしてください。
- 外部ブラウザに古い下書きが残る場合があります。TASのブラウザ保存キーは`tas_campaign_draft_v1`です。必要な下書きを保存したうえで、古い下書きを削除して再読み込みします。
