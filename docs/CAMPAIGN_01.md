## 0. スキーマ（この文書自身の構造）

```
キャンペーン
├ 1. 世界設定（World Bible）      … 全プロンプトに常時注入される不変データ
├ 2. キャスト                     … 重要NPC。三層知識モデルで記述
├ 3. 章の連なり（起承転結の設計） … 章 = quest / deliverable / world_flags
│    └ シーン = brief / goal / direction / secrets(entity) / enemy / loot / mode
└ 4. エンティティ台帳             … 正名(日英)・表層描写・ビジュアル記述子
```

システム実装との契約：**systemPrompt はこの文書のデータを注入するだけの純関数とする。
語彙レジスタや世界観の記述をコードに直書きしない**（2026-07-04「懐中電灯」事件の教訓）。
また、エンティティの③真相のうち「正体」に関わるものは、判定(check)を経由した場合のみ
開示できる。判定を経ない自由描写でLLMが正体を代わりに創作することを禁じる（D-022）。

## 1. 世界設定（World Bible）

### 1.1 世界の骨格

- 世界名：**エルドヴェイル辺境領** / The Eldervale Marches 【要議論：固有名は全て仮】
- 時代感：王道中世ファンタジー。剣と魔法、ただし魔法は稀で畏怖の対象
- 技術水準：**電気・内燃機関・火薬銃は存在しない**。光源はランタン・松明・蝋燭。
  動力は人力・畜力・水車・そして「古代の遺工（オールドワークス）」——失われた機巧技術の遺物
- 遺工（オールドワークス）：数百年前に滅びた機巧文明の残滓。自動人形・永久ランタン等。
  現代の住人には再現不能。畏怖と実利の対象。**このキャンペーンの縦軸**

### 1.2 語彙レジスタ（プロンプトに常時注入）

- 禁止語彙：現代機器（懐中電灯・電灯・モーター・エンジン・電池）、
  メートル法（→ 歩・腕尺・日程で表現）、現代口語（テンション・リスク等）
- 推奨語彙：ランタン、軌条、坑道、遺工、機巧、封鎖柵
- 文体：翻訳ファンタジー小説の文体（「〜である」調の地の文、詩的だが簡潔）
- 読解レベル：**中学生にそのまま伝わる平易な文**（ターゲット＝小学校高学年〜YA。PROJECT.md／D-017 に従う。ここで再決定しない）
  - 避ける：抽象漢語による内面・状況の評価（譲歩・逡巡・忖度・妥協 等）。
    評価語に頼らず、見えたこと・聞こえたことで示す（「語りの技法ルール」とも整合）
  - 残してよい：具象の雰囲気語彙（軌条・坑道・遺工 等）。絵が浮かぶ名詞は世界の手触りであり、文脈で学べる
- 数値の扱い：HP・DC等のゲーム数値は語りに出さない（用途契約に従う）

### 1.3 ビジュアル基調（拡張準備）

- アートスタイル：厚塗りファンタジーイラスト。劇画とアニメ調の中間 【要議論】
- 基調パレット：ランタンの橙 × 坑道の青灰（現モックUIの配色を正とする）
- 参照：design/TRPG アプリ.png、Gemini_Generated_Image_*.png
- 運用：エンティティ台帳（4章）の visual 記述子から画像生成プロンプトを組む。
  **語りの正名とアセットのキーを一致させる**（正名の原則のビジュアル適用）

### 1.4 ローカライズ方針

- 正名は**日英対**で台帳に登録する（英語版展開の布石）
- 固有名詞は英語圏で自然に響く命名を優先し、日本語は音写＋意訳の併用
  （例：Lanternhill → ランタンヒル、Wardenlight → 灯の番人）

## 3. 章の連なり（起承転結）

キャンペーン全体の問い：**「灯を守るのは、誰の仕事か」**

| 章 | 起承転結 | quest（依頼） | deliverable（運搬物） | 主なworld_flags |
|---|---|---|---|---|
| 1. 廃坑の灯 | 起 | 灯りの正体を確かめ、坑道が使えるか村に報告せよ | 番人の存在の報告 | guardian_fate（対話/撃破/回避）、learned_heartstone（心石の存在を知ったか） |
| 2. 心石の在処 | 承〜転 | （マイラの依頼が guardian_fate で変化）坑道の最深部から「石の見本」を持ち帰れ | 心石の欠片 or 持ち帰らない決断 | heartstone_choice（渡した/隠した/砕いた）、guardian_alive |
| 3. 灯の絶える夜 | 結（収束の悲劇） | 崩落が始まった坑道から村人を守れ | 生還そのもの | （終章のためフラグは持ち越さない） |

- 収束設計：どの選択でも坑道の時代は終わる（構造は収束）。
  ただし「番人がどう終わるか」「村が何を得るか」「マイラが何を暴かれるか」は
  world_flags が語りを着色する（語りは発散）
- 収束の悲劇は第3章まで温存する（設計メモ5節）。第1章では「何をしても同じ」を感じさせない
- 【要議論】標準章数：3章はMVP検証用の最小構成。GDD未決の「1キャンペーンの標準Chapter数」
  は本キャンペーンの実測（1章30分〜1時間×3）を根拠に決める

### 3.1 第1章「廃坑の灯」詳細（モックからの本編化）

- quest：マイラから受注。「灯りの正体を確かめ、坑道が使えるか報告してほしい」
  （目標は GM の②層にも注入 → 迷走時の再提示根拠）
- 報告シーン（章の締め）：マイラの反応が world_flags で分岐
  - 対話した →「番人、ですって…」身を乗り出す。②層：関心を隠しきれない演出
  - 倒した → 落胆を隠して労をねぎらう。②層：手が一瞬止まる演出
  - 心石に言及 → ②層の「抑えきれない関心」発動（第2章への伏線）

#### シーン定義（現行モックの3シーンを新スキーマで）

### 3.2 伏線台帳（設置と回収の対応表）

> 伏線 = 設置（②層で匂わせる）＋ 本体（③層の秘密）＋ 回収（開示＋強制注入）の三点セット。
> エンジンは回収期限のシーン到達時、本体が未開示なら**強制注入で回収を保証**する
> （「必須情報はシーン進行で開示を保証」ルールの伏線への拡張）。
> 状態は planted（設置シーン到達済）→ revealed（本体開示済）→ paid_off（回収描写済）と遷移。

| ID | 設置（どこで・どう匂わせるか） | 本体（③層） | 回収期限と形 | 種別 |
|---|---|---|---|---|
| F1 | 1章sc1 ②層「古び方と『最近も動く気配』の対比」 | s1b: 今も何かが軌道を使っている | 1章sc3: 番人との遭遇で自然回収（期限=遭遇時） | 装飾（ダイス開示可） |
| F2 | 1章sc1 secrets「作業札『誰も近づくな』」 | 封鎖の真意（閉じ込めではなく、出ないための封） | 2章: 封鎖柵の内側修繕の発見で**意味が反転** | 装飾→2章で必須化 |
| F3 | 1章sc3 ②層「所作のぎこちなさ・坑道の軋み」 | s3b: 心石が尽きかけ、番人が止まれば坑道は崩れる | 3章冒頭: 崩落開始の理由として**強制注入** | 必須 |
| F4 | 1章報告シーン ②層「マイラの抑えきれない関心」 | キャスト③: マイラの真の目的は心石 | 2章の依頼の変質で示唆、3章で暴露（強制注入） | 必須 |

- 種別の運用は既存の秘密分類ルールに従う：装飾=ダイス開示可、必須=進行で保証
- F2 の「意味の反転」型（同じ事実が後から違う意味になる）は伏線の最上級形。
  台帳で管理しない限り、即興の語りでは偶然にしか生まれない
- 【要議論】台帳の状態遷移をエンジン実装するのは製品段階でよい。
  モックでは F1〜F3 を第1章データに手動で埋め込み、回収の体感を検証する

シーンの mode 属性（キャンペーンと収録シナリオの細分化への回答）：
- **scripted**：全描写を事前生成テキストで再生。LLM呼び出しゼロ（無料体験版はこれのみ）
- **hybrid**：骨格は事前定義、語りは即興（現モックの動作。購入キャンペーンの標準）
- **improv**：構造も薄く即興重視（将来のユーザー制作向け。当面使わない）
